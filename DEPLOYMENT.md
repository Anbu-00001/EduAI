# EduPredict AI Deployment Guide (Free-Forever Stack)

This guide covers deploying EduPredict AI across a permanently free split-stack architecture.

## Section 1: Prerequisites
Create accounts on the following platforms:
- **Neon.tech** (PostgreSQL Database)
- **Upstash** (Redis Cache/Rate Limiting)
- **Render.com** (FastAPI Backend)
- **Vercel** (React Frontend)
- **UptimeRobot** (Keep Render alive)

## Section 2: Neon PostgreSQL Setup
1. Create a new project in Neon.
2. Note the generated Postgres connection string (`postgres://...`).
3. Connect to the database and run the schema initialization using the `app/api/schema.sql` file.

## Section 3: Upstash Redis Setup
1. Create a new Redis database in Upstash.
2. Copy the `REDIS_URL` connection string (should start with `rediss://`).

## Section 4: Render Backend Setup
1. In Render, create a new **Web Service**.
2. Connect your GitHub repository.
3. Use the **Docker** environment.
4. Render will automatically read the `render.yaml` configuration in the repository root.
5. In the Environment section on Render, you MUST manually add the following variables:
   - `DATABASE_URL`: (Paste from Neon)
   - `REDIS_URL`: (Paste from Upstash)
   - `ALLOWED_ORIGINS`: We'll update this later (temporarily set to `http://localhost:5173`)
   - `DATAGOV_API_KEY`: (Your data.gov.in API Key)

## Section 5: Vercel Frontend Setup
1. Create a new project in Vercel.
2. Connect the same GitHub repository.
3. Set the **Root Directory** to `app/ui`.
4. Vercel will automatically read the `vercel.json` and use Vite defaults.
5. In Environment Variables, add:
   - `VITE_API_BASE`: The Render backend URL (e.g., `https://edupredict-api.onrender.com/v1`)
   - `VITE_BUILD_TARGET`: `vercel`

## Section 6: Post-Deploy Steps
1. Once both services are deployed, note the frontend URL generated by Vercel (e.g., `https://edupredict-frontend.vercel.app`).
2. Go to the Render backend settings and update `ALLOWED_ORIGINS` to include the Vercel URL.
3. Use the Render shell to run the initialization script to seed the demo keys (e.g., `python scripts/seed_demo_key.py`).

## Section 7: UptimeRobot Setup
Render's free tier spins down idle instances.
1. Create an account on UptimeRobot.
2. Add a new HTTP(s) monitor pointing to your Render healthcheck URL (`https://edupredict-api.onrender.com/v1/health`).
3. Set the interval to 5 minutes to prevent cold starts.

## Section 8: Verify Deployment
Run through the following checklist to ensure everything works:
- [ ] Render Healthcheck (`/v1/health`) returns 200 OK.
- [ ] Vercel Frontend loads successfully.
- [ ] Submitting a sample assessment completes without CORS errors.
- [ ] Scheduler logs confirm it's disabled.
- [ ] Models loaded properly in `SLIM_MODEL=true` mode.
